# Pixel Community 像素社区 — 当前功能清单

> 生成日期: 2026-05-03 | 版本: v0.1

---

## 一、项目概况

2D 等轴测像素风多人在线生活模拟游戏。浏览器端 Vue3 + Canvas 渲染，Node.js/Express 后端，MongoDB + Redis，WebSocket 实时通信。

**当前可玩内容**: 注册 → 创建角色 → 进入城镇地图 → 走路/传送 → 进入挂机区域开始挂机行为 → 收集奖励升级 → 与其他在线玩家实时互动聊天

---

## 二、用户系统

### 2.1 注册
- 用户名 3-20 字符（唯一）
- 密码 ≥6 字符，bcrypt 加密
- 角色名 1-12 字符（唯一）
- 选择性别: 男/女

### 2.2 登录
- JWT Token 认证，7 天有效期
- 登录时自动结算离线期间的挂机收益（最多 100 轮）
- 登录时自动应用离线期间的属性衰减

### 2.3 角色初始属性
| 属性 | 初始值 | 范围 |
|------|--------|------|
| 精力 (energy) | 100 | 0-100 |
| 心情 (mood) | 100 | 0-100 |
| 饱腹 (hunger) | 100 | 0-100 |
| 健康 (health) | 100 | 0-100 |
| 清洁 (clean) | 100 | 0-100 |
| 力量 (strength) | 5 | 0+ |
| 智力 (intelligence) | 5 | 0+ |
| 魅力 (charm) | 5 | 0+ |
| 攻击 (attack) | 10 | 0+ |
| 防御 (defense) | 10 | 0+ |
| 速度 (speed) | 10 | 0+ |
| 闪避 (dodge) | 5 | 0+ |
| HP / 最大HP | 100 / 100 | 1+ |
| 金币 (gold) | 0 | 0+ |

## 三、属性系统

### 3.1 五大生活属性
`精力` `心情` `饱腹` `健康` `清洁` — 范围 0-100，所有值四舍五入为整数

### 3.2 属性衰减（每分钟）
| 属性 | 衰减/分钟 |
|------|-----------|
| 精力 | -0.2 |
| 饱腹 | -0.15 |
| 清洁 | -0.1 |
| 心情 | -0.05 |
| 健康 | -0.3（仅在饱腹≤0时） |

衰减在挂机结算时一次性计算（经过分钟 × 衰减率），执行挂机行为期间持续衰减。

### 3.3 三大战斗属性
`攻击` `防御` `速度` — 当前仅通过升级提升，暂无战斗系统使用

### 3.4 心情倍率系统
| 心情等级 | 心情值范围 | 收益倍率 |
|----------|-----------|---------|
| terrible | 0-20 | 0.8x |
| bad | 21-40 | 0.9x |
| normal | 41-60 | 1.0x |
| good | 61-80 | 1.1x |
| excellent | 81-100 | 1.2x |

所有挂机收益（经验、金币、属性加成）均乘以心情倍率。

## 四、等级与经验系统

### 4.1 经验曲线
- 公式: `升至N级所需经验 = floor(100 × 1.5^(N-1))`
- 最高等级: 100
- 示例: Lv1(0), Lv2(150), Lv3(225), Lv5(506), Lv10(3,841), Lv20(221,684), Lv50(1.2亿), Lv100(7.9千亿亿)
- 经验累计不清零，可连续升级

### 4.2 升级奖励（每级）
- 最大HP +5
- HP 回满
- 攻击 +2
- 防御 +2
- 速度 +1

## 五、挂机行为系统（核心玩法）

### 5.1 概述
- **5 种行为类型**: 学习(study)、打工(work)、挖矿(mining)、伐木(woodcut)、钓鱼(fishing)
- 同时只能进行 1 个挂机任务
- 开启时扣除精力，执行期间属性持续衰减
- 完成后手动收集，收益受心情倍率影响
- 离线时自动结算（最多 100 轮）
- 收集后 5 分钟内未领取则过期

## 七、地图系统

### 7.1 地图技术规格
- 2.5D 等轴测渲染（菱形瓦片，2:1 比例）
- 瓦片尺寸: 96×48px
- 建筑方块高度: 54px
- 当前 2 张地图: 城镇广场(25×25) + 房屋室内(10×10)

### 7.2 地形类型（5 种）
| 代码 | 名称 | 可行走 | 渲染方式 |
|------|------|--------|----------|
| g (grass) | 草地 | 否 | 绿色菱形 |
| r (road) | 道路 | 是 | 米色菱形 |
| i (interior) | 室内地板 | 是 | 暖色菱形 |
| b (building) | 建筑 | 否 | 3D 方块（顶面+左右面）|
| d (door) | 门口 | 是 | 3D 方块+门洞标记 |

### 7.3 地图功能
- **传送门**: 站在门口格子上自动触发地图切换（带加载过渡）
- **点击移动**: BFS 四方向寻路，角色沿路径平滑移动
- **碰撞检测**: 草地和建筑不可通行

## 十、多人实时系统（WebSocket）

### 10.1 功能
- JWT 认证的 WebSocket 连接（自动重连，指数退避）
- 按地图分房间，玩家进入地图时自动加入
- 实时广播: 玩家进入/离开/移动
- 公共聊天: 同房间所有玩家可见（自己的消息过滤不重复显示）
- 30s 心跳保活

### 10.2 消息协议
**客户端→服务端**: `ping` | `join_room` | `move` | `chat`
**服务端→客户端**: `connected` | `pong` | `room_joined` | `player_joined` | `player_left` | `player_moved` | `chat_broadcast` | `error`

## 十四、技术配置

### 14.1 后端
- Node.js + Express
- MongoDB (Mongoose ODM)
- Redis (ioredis, 已连接待使用)
- WebSocket (ws 库)
- JSON 文件配置管理 + 热重载
- 统一响应格式: `{code, message, data}`
- Docker Compose 开发环境

### 14.2 前端
- Vue3 Composition API + Pinia + Vue Router (hash 模式)
- Vite 构建 + ES Modules
- Canvas 2D 渲染引擎（自研等轴测 renderer）
- ResizeObserver 自适应画布缩放（最小宽度 640px + 8px 边距）
- Axios HTTP 客户端（JWT 自动附加 + 401 自动跳转登录）

### 14.3 已实现 API (16 个端点)
`POST /api/auth/register` `POST /api/auth/login` `GET /api/auth/me`
`GET /api/action/list` `POST /api/action/start` `GET /api/action/status` `POST /api/action/collect` `POST /api/action/cancel`
`GET /api/player/profile` `GET /api/player/attributes` `GET /api/player/inventory` `GET /api/player/skills`
`GET /api/maps` `GET /api/maps/:mapId`
`GET /api/health`
+ 4 个管理端接口 (config 热重载 + map CRUD)

## 十五、尚未实现/可扩展方向（供 AI 参考）

- **生活属性消耗闭环**: 属性会衰减但没有恢复手段（吃饭恢复饱腹、睡觉恢复精力、洗澡恢复清洁、娱乐恢复心情）
- **道具使用系统**: 背包有物品但无法使用/出售/合成
- **战斗系统**: 战斗属性+技能已定义，无 PvE/PvP 玩法
- **社交系统**: 好友、私聊、交易、公会
- **经济系统**: 商店、交易行、物品定价
- **任务系统**: 无 NPC 对话、无任务链
- **房屋系统**: 只有 1 个室内地图，无房屋建造/装修
- **技能熟练度**: 技能框架存在但升级机制未集成到行为中
- **更多地图**: 仅 2 张地图，需要扩展世界
- **手机端适配**: 目前仅桌面端
- **音效/音乐**: 无
- **成就系统**: 无
